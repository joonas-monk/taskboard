**Step 1: Update `src/lib/prisma.ts` to enable WAL mode.**

Add WAL pragma initialization BEFORE the adapter creation. The updated file should be:

```typescript
import 'dotenv/config'
import Database from 'better-sqlite3'
import { PrismaBetterSqlite3 } from '@prisma/adapter-better-sqlite3'
import { PrismaClient } from '../generated/prisma/client'

const connectionString = process.env.DATABASE_URL ?? 'file:/var/data/taskboard/taskboard.db'
const dbPath = connectionString.replace(/^file:/, '')

// Set WAL mode — persists in DB file across connections.
// Required for concurrent access from Next.js + pipeline worker process.
const rawDb = new Database(dbPath)
rawDb.pragma('journal_mode = WAL')
rawDb.close()

const adapter = new PrismaBetterSqlite3({ url: connectionString })

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma =
  globalForPrisma.prisma ?? new PrismaClient({ adapter })

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma
}
```

Key changes: Import `Database` from `better-sqlite3`, extract `dbPath` by stripping `file:` prefix, open raw connection, set WAL pragma, close raw connection. WAL mode persists in the database file — it only needs to be set once, but re-setting is idempotent and cheap.

Do NOT use `PrismaBetterSqlite3` adapter options for WAL — the adapter's constructor does not support pragma options (verified in research).
