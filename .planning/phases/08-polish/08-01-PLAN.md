After the PrismaClient is created (after `new PrismaClient({ adapter })` on line ~22) but still at module level, add an async IIFE that resets any cards with stale active pipeline status.

The logic:
1. Find all cards where `pipelineStatus` is in `['QUEUED', 'PLANNING', 'EXECUTING', 'TESTING']`
2. For each such card, update `pipelineStatus` to `'FAILED'`
3. For each such card, find the latest PipelineRun and update its `status` to `'FAILED'` and `error` to `'Palvelin kaynnistyi uudelleen -- pipeline keskeytyi'`
4. Log to console: `[TaskBoard] Palautettu ${count} keskeytettyä pipelinea FAILED-tilaan`

Use `prisma.card.findMany` + loop with individual updates (not updateMany) because we also need to update the associated PipelineRun for each card.

Wrap in try/catch -- if recovery fails, log error but do NOT prevent the app from starting.

Important: This runs once on module load. In development with hot reload, the `globalForPrisma.prisma` check prevents re-running (the prisma instance is reused). In production, it runs once on server start -- exactly the desired behavior.

The recovery code should run AFTER the prisma singleton assignment but as a fire-and-forget side effect:
